    their existing arcade game from localStorage, Google Sheets
    (GAS), or Firebase over to a self-hosted PHP API on cPanel.

    THE STAR FEATURE: A file drop zone that sends the student's
    game code to Claude via the Anthropic API. Claude detects
    which storage system is in use, finds the exact relevant
    functions, and writes ready-to-use replacement code -
    tailored to THEIR specific game, not a generic example.

    BONUS SECTION: A centralized arcade dashboard - one PHP
    file that reads all game leaderboards and displays
    cross-game analytics (plays, top scores, total time, etc.)

    Architecture used by scores.php:
      GET  scores.php?game=mygame      → returns top 10 for that game
      POST scores.php                  → saves a score for a game
        body: { game, name, score, level?, duration? }
      Stored as: scores-{gamename}.json per game

HOW IT WORKS:

POST body (JSON): { game, name, score, level, duration }
  - game:     required, string, max 40 chars, alphanumeric + hyphens only (used as filename)

SECURITY:
- header('Content-Type: application/json') as the very first output
- header('Access-Control-Allow-Origin: *') for same-server cross-file calls

- Sanitize name with htmlspecialchars, strip_tags, trim

<span class="js-cm">// ── DEFINE YOUR GAME'S UNIQUE NAME (used as the file key) ──────────
// Change "my-game" to something short that describes your game.
// Use only letters, numbers, and hyphens - no spaces.</span>
<span class="js-kw">const</span> GAME_ID = <span class="js-str">"my-game"</span>;

<span class="js-cm">// ── SAVE A SCORE TO THE PHP API ─────────────────────────────────────
// Call this function after game over instead of your old storage call.
// Parameters:
//   name     - the player's name (string)
//   score    - their final score (number)
//   level    - level they reached (number, optional)
//   duration - seconds they played this session (number, optional)</span>
<span class="js-kw">async function</span> <span class="js-fn">saveScore</span>(name, score, level = 1, duration = 0) {
  <span class="js-kw">try</span> {
    <span class="js-kw">const</span> response = <span class="js-kw">await</span> <span class="js-fn">fetch</span>(<span class="js-str">'scores.php'</span>, {
      method: <span class="js-str">'POST'</span>,
      headers: { <span class="js-str">'Content-Type'</span>: <span class="js-str">'application/json'</span> },
      body: JSON.stringify({ game: GAME_ID, name, score, level, duration })
    });
    <span class="js-kw">const</span> result = <span class="js-kw">await</span> response.json();
    <span class="js-kw">if</span> (result.error) <span class="js-kw">throw new</span> Error(result.error);
    <span class="js-kw">return</span> <span class="js-kw">true</span>;
  } <span class="js-kw">catch</span> (err) {
    console.error(<span class="js-str">'Could not save score:'</span>, err);
    <span class="js-kw">return</span> <span class="js-kw">false</span>;
  }
}

<span class="js-cm">// ── LOAD THE TOP 10 SCORES FROM THE PHP API ─────────────────────────
// Call this on page load and after a score is saved.
// Returns an array of score objects: [{name, score, level, duration, date}]</span>
<span class="js-kw">async function</span> <span class="js-fn">loadScores</span>() {
  <span class="js-kw">try</span> {
    <span class="js-kw">const</span> response = <span class="js-kw">await</span> <span class="js-fn">fetch</span>(<span class="js-str">`scores.php?game=${GAME_ID}`</span>);
    <span class="js-kw">return await</span> response.json();
  } <span class="js-kw">catch</span> (err) {
    console.error(<span class="js-str">'Could not load scores:'</span>, err);
    <span class="js-kw">return</span> [];
  }
}

    // ── HANDLE THE UPLOADED FILE ──────────────────────────────
    // Reads the file as text, then sends it to the Claude API for analysis
    function handleFile(file) {
      // Only accept HTML files
      if (!file.name.match(/\.(html|htm)$/i)) {
        showError('Please upload an .html file - your arcade game file.');
        return;
      }

      const reader = new FileReader();

      // FileReader.onload fires when the file has been read into memory
      reader.onload = (e) => {
        const code = e.target.result;
        // Start the API analysis with the file's text content
        analyzeWithClaude(file.name, code);
      };

      // readAsText reads the file as a plain string
      reader.readAsText(file);

    // ── CLAUDE API ANALYSIS ───────────────────────────────────
    // Sends the game code to the Anthropic API and asks Claude to:
    //   1. Detect which storage system is used for high scores
    //   2. Extract the relevant code snippets
    //   3. Write replacement code using fetch() → scores.php
    //   4. Generate a full migration prompt
    //
    // The response is expected as a JSON object (see prompt below).
    async function analyzeWithClaude(fileName, code) {
      // Show the loading spinner, hide everything else
      showLoading(true);
      hideError();
      hideResult();

      // Build the system prompt that instructs Claude on the expected output format.
      // We ask for strict JSON so we can parse it reliably.
      const systemPrompt = `You are a helpful coding assistant for a community college web development course. 
A student has uploaded their arcade game HTML file. Analyze it and respond with ONLY a valid JSON object - no markdown, no explanation outside the JSON, no code fences.

The JSON must have exactly these keys:
{
  "storageType": "localStorage" | "googleSheets" | "firebase" | "unknown",
  "explanation": "2-3 plain-English sentences describing what storage the game uses and where it is in the code - written for a beginner",
  "detectedCode": "the exact JavaScript functions/lines that handle saving and loading scores - copy them verbatim from the code",
  "replacementCode": "complete drop-in JavaScript replacement using fetch() to POST to scores.php and GET from scores.php?game=GAME_ID. Include a GAME_ID constant at the top. Keep the same function names as the original where possible so integration is easy. Add very descriptive comments.",
  "migrationPrompt": "a ready-to-use Claude prompt (as plain text, not JSON) that the student can paste into Claude along with their full game code, asking Claude to replace all score storage with fetch() calls to scores.php. The prompt should mention their detected storage type, include the scores.php API contract (POST body: {game, name, score, level, duration}; GET: scores.php?game=x returns top 10 array), and ask for descriptive comments throughout."
}`;

      const userMessage = `Here is my arcade game HTML file called "${fileName}". Please analyze it and return the JSON object as described.

GAME CODE:
${code.slice(0, 28000)}`; // Trim to stay within reasonable token limits

      try {
        const response = await fetch('https://api.anthropic.com/v1/messages', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            model: 'claude-sonnet-4-20250514',
            max_tokens: 1000,
            system: systemPrompt,
            messages: [{ role: 'user', content: userMessage }]
          })
        });

        // Extract the text content from Claude's response
        const data = await response.json();
        const rawText = data.content
          .filter(block => block.type === 'text')
          .map(block => block.text)
          .join('');

        // Strip any accidental markdown code fences before parsing
        const cleanJson = rawText.replace(/```json|```/g, '').trim();
        const result    = JSON.parse(cleanJson);

        // Show the result in the UI
        showResult(fileName, result);

      } catch (err) {
        // Show a user-friendly error if anything went wrong
        showError('Analysis failed. ' + (err.message || 'Please try again or paste your code into Claude directly.'));
      } finally {
        showLoading(false);
      }
    }

        p.classList.toggle('active', i === 0);
      });
    }

    // Resets the analysis section back to the drop zone
    function resetAnalysis() {
      hideResult();
      hideError();
      showLoading(false);
      // Clear the file input so the same file can be re-selected
      document.getElementById('fileInput').value = '';
    }
