<p>Everything you've built so far - HTML, CSS, JavaScript - runs in the <strong>browser</strong>. The user's computer downloads your files and runs them locally. That's called the <strong>front end</strong>.</p>
<p class="mt-2">PHP is different. It runs on the <strong>server</strong> - a computer that lives somewhere on the internet. By the time the user's browser receives anything, PHP has already done its job. That's called the <strong>back end</strong>.</p>
<!-- Visual diagram showing the request/response cycle -->

</div>
<div class="callout info mt-2"><strong><i class="callout-icon fa-solid fa-lightbulb"></i>Key idea</strong> PHP is <em>invisible</em> to the end user. By the time the browser receives the page, PHP is done - the browser only ever sees the resulting HTML. You can't "view source" and see PHP code.</div>
<!-- What PHP looks like -->
<h3 class="mt-3 mb-1">What does PHP look like?</h3>
<p>PHP files end in <code class="font-code text-purple">.php</code> instead of <code class="font-code text-purple">.html</code>. They can contain regular HTML <em>plus</em> PHP code wrapped in special tags.</p>
<div class="code-panel open" role="region" aria-label="Basic PHP syntax example"><button class="code-panel-toggle" aria-expanded="true" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-purple"></i> Example: Your first PHP file </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-comment"><!-- This is a normal HTML file ... -->
&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;&lt;title&gt;My PHP Page&lt;/title&gt;&lt;/head&gt;
&lt;body&gt;</span>

  <span class="php-comment">&lt;!-- ... but here, PHP takes over --&gt;</span>
  <span class="php-tag">&lt;?php</span>
    <span class="php-comment">// Everything between &lt;?php and ?&gt; is PHP code.</span>
    <span class="php-comment">// This line creates a variable called $greeting.</span>
    <span class="php-var">$greeting</span> <span class="php-op">=</span> <span class="php-str">"Hello, world!"</span><span class="php-op">;</span>

    <span class="php-comment">// echo sends text to the browser as HTML output.</span>
    <span class="php-kw">echo</span> <span class="php-str">"&lt;h1&gt;"</span> <span class="php-op">.</span> <span class="php-var">$greeting</span> <span class="php-op">.</span> <span class="php-str">"&lt;/h1&gt;"</span><span class="php-op">;</span>

    <span class="php-comment">// PHP can do things JS can't - like get the current server date.</span>
    <span class="php-kw">echo</span> <span class="php-str">"&lt;p&gt;Today is: "</span> <span class="php-op">.</span> <span class="php-fn">date</span>(<span class="php-str">"l, F j, Y"</span>) <span class="php-op">.</span> <span class="php-str">"&lt;/p&gt;"</span><span class="php-op">;</span>
  <span class="php-tag">?&gt;</span>

&lt;/body&gt;
&lt;/html&gt;</code></pre>
<div class="code-explanation"><strong>What's happening:</strong> The <span class="font-code text-purple">&lt;?php</span> tag starts PHP mode; <span class="font-code text-purple">?&gt;</span> ends it. <span class="font-code text-purple">$greeting</span> is a variable (PHP variables always start with <strong>$</strong>). <span class="font-code text-purple">echo</span> prints output. The <span class="font-code text-yellow">.</span> operator joins strings together (like <code>+</code> does in JavaScript).</div>
</div>
</div>
<!-- Front-end vs back-end comparison -->
<h3 class="mt-3 mb-1">JavaScript vs PHP: what runs where?</h3>
<div class="card-grid">
<div class="card">
<h3 class="text-blue">🖥️ JavaScript (Front End)</h3>
<p>Runs in the user's browser. Can update the page without reloading. Has no direct access to files or databases. Everyone can read your code.</p>
</div>
<div class="card">
<h3 class="text-purple">⚙️ PHP (Back End)</h3>
<p>Runs on the server. Can read/write files and databases. Can talk to external services. Nobody sees your PHP code - only the result.</p>
</div>
</div>
<div class="callout note mt-2"><strong><i class="callout-icon fa-solid fa-star"></i>Fun fact</strong> PHP was created in 1994 and powers approximately 77% of all websites with a known server-side language - including WordPress, Wikipedia, and Facebook (in its early years). It's one of the most widely deployed back-end technologies on the planet.</div>

<section class="section" id="what-php-can-do" aria-labelledby="what-php-can-do-heading">

<div class="section-icon green" aria-hidden="true"><i class="fa-solid fa-bolt"></i></div>
<h2 id="what-php-can-do-heading">What PHP Can Do</h2>
</div>
<p>Once code moves to the server, a whole new set of capabilities opens up. Here are the most important things PHP can do that front-end code simply can't.</p>
<!-- ---- Capability 1: Handle Forms ---- -->
<h3 class="mt-3 mb-1 text-green">1. Handle Form Submissions</h3>
<p>When a user fills out a form and clicks Submit, the data travels to the server as an HTTP request. PHP reads that data, processes it (validates, saves, emails, etc.) and sends back a response. This is the foundation of almost every web app.</p>
<div class="code-panel" role="region" aria-label="Form handling example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-green"></i> Example: Reading a submitted form </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-comment">// $_POST is a special PHP "superglobal" - it holds data</span>
  <span class="php-comment">// that was submitted via an HTML form with method="POST".</span>

  <span class="php-comment">// isset() checks whether a variable exists before we use it.</span>
  <span class="php-comment">// This prevents errors if the page loads without a form submission.</span>
  <span class="php-kw">if</span> (<span class="php-fn">isset</span>(<span class="php-var">$_POST</span>[<span class="php-str">'name'</span>])) {

    <span class="php-comment">// htmlspecialchars() converts special characters to HTML-safe ones.</span>
    <span class="php-comment">// ALWAYS do this before displaying user input - it prevents attacks.</span>
    <span class="php-var">$name</span> <span class="php-op">=</span> <span class="php-fn">htmlspecialchars</span>(<span class="php-var">$_POST</span>[<span class="php-str">'name'</span>])<span class="php-op">;</span>

    <span class="php-kw">echo</span> <span class="php-str">"&lt;p&gt;Nice to meet you, "</span> <span class="php-op">.</span> <span class="php-var">$name</span> <span class="php-op">.</span> <span class="php-str">"!&lt;/p&gt;"</span><span class="php-op">;</span>
  }
<span class="php-tag">?&gt;</span>

<span class="php-comment">&lt;!-- The HTML form that sends data to this same PHP file --&gt;</span>
&lt;form method="POST" action=""&gt;
  &lt;input type="text" name="name" placeholder="Your name" /&gt;
  &lt;button type="submit"&gt;Say Hello&lt;/button&gt;
&lt;/form&gt;</code></pre>
<div class="code-explanation"><strong>Key concepts:</strong> <strong>$_POST</strong> holds form data. Setting <strong>action=""</strong> on the form means it submits to the same PHP file. Notice <strong>htmlspecialchars()</strong> - that's your first security tool, and we'll talk about why it matters in the Security section.</div>
</div>
</div>
<!-- ---- Capability 2: Read/Write Files ---- -->
<h3 class="mt-3 mb-1 text-green">2. Read and Write Files</h3>
<p>PHP can create, read, update, and delete files on the server. This lets you persist data without a full database - great for simple apps like a guestbook, a log file, or saving user-submitted content.</p>
<div class="code-panel" role="region" aria-label="File writing example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-green"></i> Example: Writing and reading a text file </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-comment">// Define where the file will live on the server.</span>
  <span class="php-comment">// __DIR__ means "the same folder as this PHP file" - safer than a relative path.</span>
  <span class="php-var">$file_path</span> <span class="php-op">=</span> <span class="php-fn">__DIR__</span> <span class="php-op">.</span> <span class="php-str">'/messages.txt'</span><span class="php-op">;</span>

  <span class="php-comment">// If a form was submitted, save the message to the file.</span>
  <span class="php-kw">if</span> (<span class="php-fn">isset</span>(<span class="php-var">$_POST</span>[<span class="php-str">'message'</span>])) {

    <span class="php-comment">// Clean the input before saving it. Never trust user data!</span>
    <span class="php-var">$message</span> <span class="php-op">=</span> <span class="php-fn">htmlspecialchars</span>(<span class="php-fn">trim</span>(<span class="php-var">$_POST</span>[<span class="php-str">'message'</span>]))<span class="php-op">;</span>

    <span class="php-comment">// file_put_contents() writes text to a file.</span>
    <span class="php-comment">// FILE_APPEND adds to the end instead of overwriting.</span>
    <span class="php-fn">file_put_contents</span>(<span class="php-var">$file_path</span>, <span class="php-var">$message</span> <span class="php-op">.</span> <span class="php-str">"\n"</span>, <span class="php-kw">FILE_APPEND</span>)<span class="php-op">;</span>
  }

  <span class="php-comment">// Read all saved messages from the file (returns false if it doesn't exist yet).</span>
  <span class="php-var">$all_messages</span> <span class="php-op">=</span> <span class="php-fn">file</span>(<span class="php-var">$file_path</span>, <span class="php-kw">FILE_IGNORE_NEW_LINES</span> <span class="php-op">|</span> <span class="php-kw">FILE_SKIP_EMPTY_LINES</span>)<span class="php-op">;</span>

  <span class="php-comment">// Loop through each message and print it as a list item.</span>
  <span class="php-kw">if</span> (<span class="php-var">$all_messages</span>) {
    <span class="php-kw">echo</span> <span class="php-str">"&lt;ul&gt;"</span><span class="php-op">;</span>
    <span class="php-kw">foreach</span> (<span class="php-var">$all_messages</span> <span class="php-kw">as</span> <span class="php-var">$msg</span>) {
      <span class="php-kw">echo</span> <span class="php-str">"&lt;li&gt;"</span> <span class="php-op">.</span> <span class="php-var">$msg</span> <span class="php-op">.</span> <span class="php-str">"&lt;/li&gt;"</span><span class="php-op">;</span>
    }
    <span class="php-kw">echo</span> <span class="php-str">"&lt;/ul&gt;"</span><span class="php-op">;</span>
  }
<span class="php-tag">?&gt;</span></code></pre>
<div class="code-explanation"><strong>Key concepts:</strong> <strong>file_put_contents()</strong> writes to a file. <strong>FILE_APPEND</strong> adds to the existing file rather than replacing it. <strong>file()</strong> reads the file back as an array of lines. <strong>foreach</strong> loops through each item - just like <code>for...of</code> in JavaScript.</div>
</div>
</div>
<!-- ---- Capability 3: Return JSON for fetch() ---- -->
<h3 class="mt-3 mb-1 text-green">3. Act as an API (Return JSON)</h3>
<p>PHP can respond with <strong>JSON data</strong> instead of HTML. This is a game-changer: your front-end JavaScript can use <code class="font-code">fetch()</code> to call a PHP file and get structured data back - just like calling a third-party API, except <em>you control it</em>.</p>
<div class="code-panel" role="region" aria-label="PHP as JSON API example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-green"></i> Example: PHP file that returns JSON </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-comment">// api.php - This file acts like a mini API endpoint.</span>
<span class="php-tag">&lt;?php</span>
  <span class="php-comment">// Tell the browser this response is JSON, not HTML.</span>
  <span class="php-fn">header</span>(<span class="php-str">'Content-Type: application/json'</span>)<span class="php-op">;</span>

  <span class="php-comment">// Build an array of data (just like an object in JavaScript).</span>
  <span class="php-var">$data</span> <span class="php-op">=</span> [
    <span class="php-str">'status'</span>   <span class="php-op">=&gt;</span> <span class="php-str">'ok'</span>,
    <span class="php-str">'time'</span>     <span class="php-op">=&gt;</span> <span class="php-fn">date</span>(<span class="php-str">'H:i:s'</span>),
    <span class="php-str">'message'</span>  <span class="php-op">=&gt;</span> <span class="php-str">'Hello from the server!'</span>,
  ]<span class="php-op">;</span>

  <span class="php-comment">// json_encode() converts the PHP array into a JSON string.</span>
  <span class="php-kw">echo</span> <span class="php-fn">json_encode</span>(<span class="php-var">$data</span>)<span class="php-op">;</span>
<span class="php-tag">?&gt;</span>

<span class="php-comment">// ----- In your HTML/JS file, call this with fetch(): -----</span>

<span class="php-comment">// fetch('api.php')               → makes a GET request to your PHP file</span>
<span class="php-comment">// .then(res =&gt; res.json())       → parse the JSON response</span>
<span class="php-comment">// .then(data =&gt; console.log(data)) → use the data however you want</span></code></pre>
<div class="code-explanation"><strong>Key concept:</strong> By setting the Content-Type header to <strong>application/json</strong> and using <strong>json_encode()</strong>, a PHP file becomes a proper API endpoint. Your front-end JavaScript doesn't need to know it's PHP - it just calls the URL and gets JSON back, exactly like any other API.</div>
</div>
</div>
<!-- ---- Capability 4: Sessions ---- -->
<h3 class="mt-3 mb-1 text-green">4. Remember Users with Sessions</h3>
<p>HTTP is "stateless" - each page request is forgotten immediately. PHP <strong>sessions</strong> let you remember information about a user across multiple page loads, like whether they're logged in or what's in their cart.</p>
<div class="code-panel" role="region" aria-label="Sessions example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-green"></i> Example: Counting page visits with sessions </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-comment">// session_start() MUST be the very first thing in any PHP file that uses sessions.</span>
  <span class="php-comment">// It loads any existing session data for this visitor.</span>
  <span class="php-fn">session_start</span>()<span class="php-op">;</span>

  <span class="php-comment">// $_SESSION is a special array that persists across page loads.</span>
  <span class="php-comment">// If 'visit_count' doesn't exist yet, set it to 0.</span>
  <span class="php-kw">if</span> (!<span class="php-fn">isset</span>(<span class="php-var">$_SESSION</span>[<span class="php-str">'visit_count'</span>])) {
    <span class="php-var">$_SESSION</span>[<span class="php-str">'visit_count'</span>] <span class="php-op">=</span> <span class="php-num">0</span><span class="php-op">;</span>
  }

  <span class="php-comment">// Increment the count each time this page loads.</span>
  <span class="php-var">$_SESSION</span>[<span class="php-str">'visit_count'</span>]<span class="php-op">++;</span>

  <span class="php-kw">echo</span> <span class="php-str">"You have visited this page "</span>
     <span class="php-op">.</span> <span class="php-var">$_SESSION</span>[<span class="php-str">'visit_count'</span>]
     <span class="php-op">.</span> <span class="php-str">" time(s) this session."</span><span class="php-op">;</span>
<span class="php-tag">?&gt;</span></code></pre>
<div class="code-explanation"><strong>Key concept:</strong> Sessions store data on the <em>server</em> and give the browser a cookie with a session ID. Unlike <strong>localStorage</strong> (which stores data in the browser), session data is never sent to the user directly - they only see an ID.</div>
</div>
</div>
<!-- ---- Capability 5: Send Email ---- -->
<h3 class="mt-3 mb-1 text-green">5. Send Email</h3>
<p>PHP has a built-in <code class="font-code text-yellow">mail()</code> function, and cPanel accounts usually have a mail server already configured. Combined with a form, this powers the classic "contact us" feature that every website needs.</p>
<div class="code-panel" role="region" aria-label="Send mail example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-green"></i> Example: Sending a contact form email </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-kw">if</span> (<span class="php-var">$_SERVER</span>[<span class="php-str">'REQUEST_METHOD'</span>] <span class="php-op">===</span> <span class="php-str">'POST'</span>) {

    <span class="php-comment">// Clean every piece of user input before using it.</span>
    <span class="php-var">$name</span>    <span class="php-op">=</span> <span class="php-fn">htmlspecialchars</span>(<span class="php-fn">trim</span>(<span class="php-var">$_POST</span>[<span class="php-str">'name'</span>] <span class="php-op">??</span> <span class="php-str">''</span>))<span class="php-op">;</span>
    <span class="php-var">$email</span>   <span class="php-op">=</span> <span class="php-fn">filter_var</span>(<span class="php-var">$_POST</span>[<span class="php-str">'email'</span>] <span class="php-op">??</span> <span class="php-str">''</span>, <span class="php-kw">FILTER_SANITIZE_EMAIL</span>)<span class="php-op">;</span>
    <span class="php-var">$message</span> <span class="php-op">=</span> <span class="php-fn">htmlspecialchars</span>(<span class="php-fn">trim</span>(<span class="php-var">$_POST</span>[<span class="php-str">'message'</span>] <span class="php-op">??</span> <span class="php-str">''</span>))<span class="php-op">;</span>

    <span class="php-comment">// Validate: make sure the email address looks real.</span>
    <span class="php-kw">if</span> (<span class="php-fn">filter_var</span>(<span class="php-var">$email</span>, <span class="php-kw">FILTER_VALIDATE_EMAIL</span>)) {

      <span class="php-var">$to</span>      <span class="php-op">=</span> <span class="php-str">'you@yourdomain.com'</span><span class="php-op">;</span>
      <span class="php-var">$subject</span> <span class="php-op">=</span> <span class="php-str">'Contact Form: '</span> <span class="php-op">.</span> <span class="php-var">$name</span><span class="php-op">;</span>
      <span class="php-var">$body</span>    <span class="php-op">=</span> <span class="php-str">"From: $name\nEmail: $email\n\n$message"</span><span class="php-op">;</span>
      <span class="php-var">$headers</span> <span class="php-op">=</span> <span class="php-str">"From: noreply@yourdomain.com"</span><span class="php-op">;</span>

      <span class="php-fn">mail</span>(<span class="php-var">$to</span>, <span class="php-var">$subject</span>, <span class="php-var">$body</span>, <span class="php-var">$headers</span>)<span class="php-op">;</span>
      <span class="php-kw">echo</span> <span class="php-str">"&lt;p&gt;✅ Message sent!&lt;/p&gt;"</span><span class="php-op">;</span>

    } <span class="php-kw">else</span> {
      <span class="php-kw">echo</span> <span class="php-str">"&lt;p&gt;❌ Please enter a valid email address.&lt;/p&gt;"</span><span class="php-op">;</span>
    }
  }
<span class="php-tag">?&gt;</span></code></pre>
<div class="code-explanation"><strong>Key concepts:</strong> <strong>filter_var()</strong> with <strong>FILTER_VALIDATE_EMAIL</strong> checks that the email address looks legitimate before we use it. The <strong>??</strong> operator provides a default value if the field is missing - this prevents PHP warnings. Always validate before you act.</div>
</div>

<h2 id="security-heading">Security Considerations</h2>
</div>
<p>The moment you write code that runs on a server - code that can read files, talk to databases, or send email - security goes from a nice-to-have to a <strong>must-have</strong>. The good news: a handful of consistent habits will protect you from the most common attacks.</p>
<div class="callout warning"><strong><i class="callout-icon fa-solid fa-triangle-exclamation"></i>Non-negotiable</strong> Security isn't something you add after an app works. It's built in from the very first line. When you ask Claude to write PHP, always include security requirements in your prompt.</div>
<!-- ---- Threat 1: XSS ---- -->
<h3 class="mt-3 mb-1 text-red">⚠️ Cross-Site Scripting (XSS)</h3>
<p>If you display user input directly on a page without sanitizing it, a bad actor can submit JavaScript code as their "name" and have it run in every visitor's browser. This is called a Cross-Site Scripting attack, or XSS.</p>
<table class="compare-table" aria-label="XSS prevention: do vs don't">
<thead>
<tr>
<th class="dont">❌ Vulnerable (never do this)</th>
<th class="do">✅ Safe (always do this)</th>
</tr>
</thead>
<tbody>
<tr>
<td class="dont">echo $_POST['name'];</td>
<td class="do">echo htmlspecialchars($_POST['name']);</td>
</tr>
<tr>
<td class="dont">echo "Hello " . $name;</td>
<td class="do">echo "Hello " . htmlspecialchars($name);</td>
</tr>
</tbody>
</table>
<div class="callout info"><strong><i class="callout-icon fa-solid fa-circle-info"></i>What htmlspecialchars() does</strong> It converts characters like <code>&lt;</code>, <code>&gt;</code>, and <code>"</code> into harmless HTML entities. So if someone submits <code>&lt;script&gt;alert('hacked')&lt;/script&gt;</code> as their name, it gets displayed as literal text - not executed as code.</div>
<!-- ---- Threat 2: SQL Injection ---- -->
<h3 class="mt-3 mb-1 text-red">⚠️ SQL Injection</h3>
<p>If your PHP talks to a database (you'll do this in a later unit), user input must <strong>never</strong> be placed directly into a database query. An attacker can craft input that changes the meaning of your query and steal or delete data.</p>
<div class="code-panel" role="region" aria-label="SQL injection example"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-red"></i> Why SQL injection is dangerous (and how to prevent it) </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-comment">// ❌ VULNERABLE: User input goes directly into the query string.</span>
<span class="php-comment">// If someone enters:  ' OR '1'='1  as the username...</span>
<span class="php-comment">// ...the query becomes: SELECT * FROM users WHERE username = '' OR '1'='1'</span>
<span class="php-comment">// ...which returns ALL users. This is SQL injection.</span>

<span class="php-var">$username</span> <span class="php-op">=</span> <span class="php-var">$_POST</span>[<span class="php-str">'username'</span>]<span class="php-op">;</span>
<span class="php-var">$query</span>    <span class="php-op">=</span> <span class="php-str">"SELECT * FROM users WHERE username = '$username'"</span><span class="php-op">;</span>
<span class="php-comment">// NEVER do this ^</span>


<span class="php-comment">// ✅ SAFE: Use PDO Prepared Statements.</span>
<span class="php-comment">// The ? is a placeholder. The actual value is sent separately,</span>
<span class="php-comment">// so the database engine knows what is data and what is code.</span>

<span class="php-var">$stmt</span> <span class="php-op">=</span> <span class="php-var">$pdo</span><span class="php-op">-&gt;</span><span class="php-fn">prepare</span>(<span class="php-str">"SELECT * FROM users WHERE username = ?"</span>)<span class="php-op">;</span>
<span class="php-var">$stmt</span><span class="php-op">-&gt;</span><span class="php-fn">execute</span>([ <span class="php-var">$_POST</span>[<span class="php-str">'username'</span>] ])<span class="php-op">;</span>
<span class="php-var">$user</span> <span class="php-op">=</span> <span class="php-var">$stmt</span><span class="php-op">-&gt;</span><span class="php-fn">fetch</span>()<span class="php-op">;</span>
<span class="php-comment">// Always use prepared statements when touching a database.</span></code></pre>
<div class="code-explanation"><strong>The rule:</strong> Never put <strong>any</strong> user-supplied value directly into a SQL query string. Always use <strong>prepared statements</strong> with placeholders. Claude knows how to do this - just tell it "use prepared statements" in your prompt.</div>
</div>
</div>
<!-- ---- Threat 3: Error Messages ---- -->
<h3 class="mt-3 mb-1 text-red">⚠️ Don't Show Error Messages to Users</h3>
<p>Detailed PHP errors are incredibly useful during development - but if they appear on a live site, they hand attackers a map of your code, file paths, and database structure.</p>
<div class="code-panel" role="region" aria-label="Error display settings"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-red"></i> Development vs Production error settings </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-comment">// ✅ DEVELOPMENT: Turn errors on while you're building.</span>
  <span class="php-comment">// You want to see everything that goes wrong during development.</span>
  <span class="php-fn">ini_set</span>(<span class="php-str">'display_errors'</span>, <span class="php-num">1</span>)<span class="php-op">;</span>
  <span class="php-fn">error_reporting</span>(<span class="php-kw">E_ALL</span>)<span class="php-op">;</span>

  <span class="php-comment">// ❌ On a live / public site, switch to this instead:</span>
  <span class="php-comment">// Errors are hidden from users but still logged to a file you can check.</span>
  <span class="php-fn">ini_set</span>(<span class="php-str">'display_errors'</span>, <span class="php-num">0</span>)<span class="php-op">;</span>
  <span class="php-fn">ini_set</span>(<span class="php-str">'log_errors'</span>, <span class="php-num">1</span>)<span class="php-op">;</span>
  <span class="php-fn">ini_set</span>(<span class="php-str">'error_log'</span>, <span class="php-fn">__DIR__</span> <span class="php-op">.</span> <span class="php-str">'/error.log'</span>)<span class="php-op">;</span>
<span class="php-tag">?&gt;</span></code></pre>
<div class="code-explanation"><strong>The rule:</strong> Log errors to a file, never display them to visitors. cPanel has a built-in Error Log viewer in the "Logs" section - check that instead of displaying errors publicly.</div>
</div>
</div>
<!-- ---- Threat 4: File Uploads ---- -->
<h3 class="mt-3 mb-1 text-red">⚠️ Be Careful With File Uploads</h3>
<p>Letting users upload files is a powerful feature - and a major risk if not handled carefully. An attacker might try to upload a PHP file disguised as an image, then execute it on your server.</p>
<div class="card-grid mt-2">
<div class="card">
<h3 class="text-green">✅ Safe practices</h3>
<p>Check file extension AND MIME type. Rename uploaded files (never use the original name). Store files outside the web root if possible. Limit file size.</p>
</div>
<div class="card">
<h3 class="text-red">❌ What to avoid</h3>
<p>Never trust <code>$_FILES['type']</code> alone - it can be faked. Never allow PHP, HTML, or executable file types to be uploaded.</p>
</div>
</div>
<!-- ---- Input Validation Principle ---- -->
<h3 class="mt-3 mb-1 text-red">⚠️ The Golden Rule: Never Trust User Input</h3>
<div class="callout warning"><strong><i class="callout-icon fa-solid fa-lock"></i>The Golden Rule</strong> Every piece of data that comes from the outside world - form fields, URL parameters, cookies, uploaded files - must be treated as potentially hostile until you have validated and sanitized it. <em>Every time. No exceptions.</em></div>
<div class="code-panel" role="region" aria-label="PHP validation tools"><button class="code-panel-toggle" aria-expanded="false" onclick="togglePanel(this)"> <span class="toggle-label"> <i class="fa-solid fa-code text-red"></i> PHP's built-in validation toolkit </span> <i class="fa-solid fa-chevron-down chevron" aria-hidden="true"></i> </button>
<div class="code-panel-body">
<pre class="code-block"><code><span class="php-tag">&lt;?php</span>
  <span class="php-comment">// PHP has filter_var() - your best friend for validation.</span>

  <span class="php-comment">// Validate an email address:</span>
  <span class="php-fn">filter_var</span>(<span class="php-var">$email</span>, <span class="php-kw">FILTER_VALIDATE_EMAIL</span>)<span class="php-op">;</span>

  <span class="php-comment">// Validate that something is a number:</span>
  <span class="php-fn">filter_var</span>(<span class="php-var">$age</span>, <span class="php-kw">FILTER_VALIDATE_INT</span>)<span class="php-op">;</span>

  <span class="php-comment">// Validate a URL:</span>
  <span class="php-fn">filter_var</span>(<span class="php-var">$url</span>, <span class="php-kw">FILTER_VALIDATE_URL</span>)<span class="php-op">;</span>

  <span class="php-comment">// Sanitize (clean up) a string - remove HTML tags:</span>
  <span class="php-fn">strip_tags</span>(<span class="php-var">$input</span>)<span class="php-op">;</span>

  <span class="php-comment">// Sanitize and escape for HTML output:</span>
  <span class="php-fn">htmlspecialchars</span>(<span class="php-var">$input</span>, <span class="php-kw">ENT_QUOTES</span>, <span class="php-str">'UTF-8'</span>)<span class="php-op">;</span>

  <span class="php-comment">// Trim whitespace from both ends of user input:</span>
  <span class="php-fn">trim</span>(<span class="php-var">$input</span>)<span class="php-op">;</span>

  <span class="php-comment">// Check a field isn't empty before using it:</span>
  <span class="php-kw">if</span> (<span class="php-fn">empty</span>(<span class="php-var">$input</span>)) {
    <span class="php-kw">echo</span> <span class="php-str">"This field is required."</span><span class="php-op">;</span>
  }
<span class="php-tag">?&gt;</span></code></pre>
<div class="code-explanation"><strong>Your standard checklist:</strong> For every piece of user input: (1) <strong>trim()</strong> to remove whitespace, (2) <strong>validate</strong> the format, (3) <strong>sanitize/escape</strong> before using it in output or queries. When you prompt Claude, ask for all three.</div>
</div>
</div>

    // FUNCTION: togglePanel(button)
    //
    // This function is called when a student clicks a code panel

      // Walk up the DOM from the button to find the parent .code-panel div

      const isOpen = panel.classList.toggle('open');

      // Update the aria-expanded attribute for screen readers.

    //   - The button briefly shows "Copied!" with green styling

    //   button - the "Copy" <button> that was clicked

        button.textContent = 'Select text manually';
        setTimeout(() => { button.textContent = 'Copy'; }, 2500);
      });
    }
